HTTP requests

Components to send HTTP/HTPPS requests.

<http|post>

Issues HTTP GET or POST request accordingly.

<!-- invoke "test" action on a "/api/public" endpoint -->
<all>
    <!-- define message to send -->
    <json>{"message":{"type":"test", "payload":"payload"}}</json>
    <!-- send post request to the URL specified, put reply to "data" variable (parsing received strings as a JSON)-->
    <post url="http://localhost:1895/api/public" to="data" json="true">
        <!-- get an object in a "message" variable, serialize and send it as "data" form value -->
        <data path="message" name="data"/>
    </post>
</all>
  • json a boolean flag to parse a request reply as a JSON object
  • path a namespace path to an object to send with a request
  • url a URL to issue requests to, is used when no "url-path" value was found
  • url-path a namespace path to URL, when given is used instead of "url" attribute
  • to a namespace path of the reply
  • https

    to configure HTTPS connections

    • skip-verify a boolean flag which controls whether a client verifies the server's certificate chain and host name
  • proxy

    proxy configuration options

    • address a proxy address
    • username a user name for proxy authentication
    • password a password for proxy authentication
  • data

    to configure form values sent

    • name a name of the form value
    • path a path to the value to send
