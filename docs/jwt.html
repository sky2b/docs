JWT

Components to sign, parse and generate keys for JSON Web Tokens (JWT).

<jwt>

The main components to work with JWT. For example you can parse token in a string: 

<!--
Parse a JWT token, for which we need only a public key.
The key could be found in the "keys" directory. It should not be generated here.
-->
<jwt dir="keys" generate="false">
    <!-- file containing public key -->
    <public file="jwt.pub"/>
    <!-- parse the string which could be found in a "token" field of the user request -->
    <parse path="data.token" to="token">
        <!-- in case of any error, we may just return a JSON object with an error description -->
        <error>
            <json>{"reply":{"error":"not authenticated"}}</json>
        </error>
    </parse>
</jwt>

Or you could sign an object and create a token: 

<!--
sign using JWT keys found in "keys" directory,
generate new keys if no keys found
-->
<jwt dir="keys" generate="true">
	<!-- private key (to sign JWT token ) -->
	<private file="jwt.pri"/>
	<!--
	sign "token" object and place generated JWT
	to "reply.token" path
	-->
	<sign path="token" to="reply.token"/>
</jwt>
  • dir directory containing key files
  • generate a boolean flag to generate private and public keys
  • public

    configures public key

    • file a file name of the file containing a public key (in a directory specified in component)
  • private

    configures private key

    • file a file name of the file containing a private key (in a directory specified in component)
    • key-bits RSA key length in bits (4096 by default)
  • sign

    signs a specified object using a private key to create a new JWT token

    • path a namespace path to the object to be singed
    • to a namespace path for the generated JWT token
    • timeout timeout (in hours)
  • parse

    parse JWT tokens and extract claims they contain

    • path a namespace path of the token
    • to a namespace path where the parsed token object will be extracted
    • then container for components executed after the token was successfully extracted
    • error container for components executed after the token was successfully extracted
